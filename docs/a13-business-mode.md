# A13 业务模式与功能闭环

## 1. 面向对象

系统主要服务 3 类角色：

1. 学生
   - 用于职业探索、目标岗位选择、差距分析和成长规划
2. 教师
   - 用于查看学生画像、职业报告和跟踪建议
3. 管理员
   - 用于维护岗位数据、知识库和系统运行状态

## 2. 业务闭环

本项目不是单点的岗位推荐工具，而是一个完整的职业规划闭环：

1. 岗位理解
   - 从企业岗位数据中清洗并构建岗位知识库
   - 形成岗位画像和岗位图谱
2. 学生理解
   - 采集简历、证书、技能和经历
   - 生成学生画像
3. 智能匹配
   - 推荐当前较匹配岗位
   - 支持学生主动选择理想岗位
4. 差距分析
   - 对比学生当前能力与目标岗位要求
   - 找出技能、证书、专业方向和经历差距
5. 路径规划
   - 生成学习建议、实践任务和阶段目标
   - 给出晋升路径与换岗路径
6. 报告输出
   - 生成职业发展报告
   - 支持教师查看与学生复盘
7. 动态迭代
   - 学生能力变化后再次评估
   - 形成“规划 -> 执行 -> 反馈 -> 再规划”的闭环

## 3. 对学生的价值

1. 从“我能投什么岗”变成“我适合什么岗、差什么、怎么补”
2. 从一次性推荐变成持续跟踪式成长规划
3. 同时支持低年级职业探索和高年级求职匹配

## 4. 对教师的价值

1. 教师能够快速看到学生的能力画像与岗位适配度
2. 教师能够基于系统结果做有针对性的点拨
3. 教师能够围绕报告结果开展后续跟踪和复盘

## 5. 对比赛展示的价值

这一页在答辩里可以直接回答 3 个问题：

1. 你们做的到底是职业规划还是岗位推荐
   - 答：二者兼备，但核心是职业规划闭环
2. 为什么需要简历和学生画像
   - 答：因为系统要判断学生当前能力，再做岗位推荐和差距分析
3. 为什么这个项目不是普通问答机器人
   - 答：因为系统有岗位知识库、岗位画像、匹配分析、路径规划和报告输出，形成了完整业务链路
