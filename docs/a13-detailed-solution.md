# A13 赛题详细方案

## 1. 项目概述

项目名称：CareerPilot  
赛题方向：基于 AI 的大学生职业规划智能体

本项目围绕 A13 赛题要求，构建一个面向大学生职业规划与人岗匹配的智能体系统。系统以岗位知识库、学生画像、人岗匹配、职业路径规划和发展报告生成为核心能力，支持从大一到大四不同阶段学生的职业探索、能力诊断与成长建议。

## 2. 需求分析

### 2.1 业务目标

系统需要解决 3 类核心问题：

1. 岗位理解
   - 从企业岗位数据中提取岗位能力要求
   - 对相同岗位进行聚合，形成岗位画像
   - 建立岗位之间的晋升与转岗路径
2. 学生理解
   - 从简历、证书、手工填写信息中抽取学生能力信息
   - 形成学生就业能力画像
3. 决策支持
   - 判断学生和岗位之间的匹配情况
   - 给出目标岗位差距分析、学习路径和阶段规划
   - 生成可编辑、可导出的职业发展报告

### 2.2 对齐企业答疑的关键约束

1. 岗位库必须以企业提供数据为基础，并按计算机/信息类岗位口径清洗。
2. 岗位画像是“相同岗位合并后的结果”。
3. 人岗匹配与目标岗位差距分析要同时具备。
4. 报告中要体现“当前较匹配岗位”和“理想岗位”。
5. 本地知识库资料需要导出为可提交的 JSON。
6. 评测需要贴近企业给出的 80% / 90% 准确率口径。

## 3. 数据方案

### 3.1 岗位数据来源

当前系统使用企业官方岗位表作为基础数据源，并补充扩展岗位数据：

1. 官方原始表：`9958` 条
2. 严格清洗后官方可用岗位：`3906` 条
3. 严格清洗后官方唯一岗位：`3430` 条
4. 增量补抓并合并后唯一岗位：`6860` 条

### 3.2 清洗原则

岗位清洗遵循以下规则：

1. 只保留计算机/信息类相关岗位
2. 允许互联网、通信、电子设备、人工智能等行业下的信息类岗位保留
3. 对明显偏销售、客服、纯行政的岗位做剔除
4. 不因链接失效删除记录，重点使用岗位表字段
5. 允许部分字段缺失，通过同岗位多条记录聚合补齐

### 3.3 岗位聚合方式

系统不按单条招聘记录直接展示岗位画像，而是：

1. 对同类岗位做标题归一化
2. 聚合同岗位的技能、学历、证书、职责描述
3. 生成统一岗位画像
4. 将岗位画像同步进入知识库与岗位图谱

## 4. 系统设计

### 4.1 总体架构

系统按“数据层 + 智能层 + 应用层”设计：

1. 数据层
   - 岗位数据导入与清洗
   - 本地知识库导出
   - 岗位图谱构建
2. 智能层
   - 岗位画像生成
   - 学生画像生成
   - 人岗匹配分析
   - 职业路径规划
   - 报告生成与润色
3. 应用层
   - 学生端
   - 教师端
   - 管理端

### 4.2 核心模块

1. 岗位知识库模块
   - 导入官方岗位表
   - 清洗与补抓岗位
   - 生成岗位画像
   - 导出本地知识库 JSON
2. 学生画像模块
   - 支持简历与材料上传
   - 支持手工录入学生信息
   - 形成学生能力画像
3. 匹配分析模块
   - 输出总分、分维度得分、优势、差距与建议
   - 同时支持推荐岗位与目标岗位差距分析
4. 职业路径模块
   - 输出晋升路径
   - 输出换岗路径
   - 输出能力提升建议
5. 报告模块
   - 生成职业发展报告
   - 支持完整性检查、智能润色、手动编辑与导出

## 5. 核心流程

### 5.1 岗位库构建流程

1. 导入企业官方岗位数据
2. 按计算机/信息类口径清洗
3. 补抓相关岗位并合并
4. 聚合同岗位记录
5. 生成岗位画像
6. 同步知识库与图谱

### 5.2 学生分析流程

1. 学生上传简历或手动填写资料
2. 系统抽取学历、专业、证书、技能、经历
3. 生成学生画像
4. 进入人岗匹配与目标岗位分析

### 5.3 人岗匹配与规划流程

1. 计算学生与岗位画像的匹配度
2. 输出当前较匹配岗位
3. 如果学生选择理想岗位，则同时输出差距分析
4. 基于岗位图谱生成成长路径
5. 生成学习计划与阶段性评估指标
6. 输出职业发展报告
